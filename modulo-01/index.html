Estructura

📝 Fundamentos de Dart para GIS

Dart es el lenguaje de Flutter. Solo aprenderemos lo que necesitamos para la app de mapas.

📦 Variables — Guardar información geográfica

Las variables son contenedores con nombre. En GIS trabajas con coordenadas, nombres de capas, escalas — todo eso se guarda en variables.

Dart
// String — texto
String nombreCapa = 'Ríos principales';
String proyeccion = 'EPSG:4326';

// int — números enteros
int zoom = 15;
int totalPuntos = 42;

// double — números decimales (¡coordenadas!)
double latitud = -16.5000;
double longitud = -68.1500;

// bool — verdadero o falso
bool capaVisible = true;
bool gpsActivo = false;
⚠️
Punto clave: Las coordenadas siempre son double, nunca int. La Paz está en -16.5, no en -16.
Las variables son como los campos de una tabla de atributos. El nombre del campo es el nombre de la variable, y el tipo (String, int, double) es como elegir "Texto", "Entero" o "Decimal" al crear un campo.
🔒 var vs final — ¿Puede cambiar o no?
Dart
// var: puede cambiar después
var zoomActual = 10;
zoomActual = 15; // ✅ OK

// final: se asigna una vez y NO cambia
final String nombreApp = 'GeoCollect';
// nombreApp = 'Otra'; // ❌ ERROR
💡
Si cambia (zoom, posición, capa activa) → var. Si nunca cambia (nombre de app, URL del servidor) → final.
📋 List — Colecciones de datos
Dart
List<String> capas = ['Ríos', 'Carreteras', 'Municipios'];

capas.add('Lagos');              // Agregar
String primera = capas[0];       // Acceder (empieza en 0)
int total = capas.length;         // Cuántos hay: 4

// Recorrer toda la lista
for (String capa in capas) {
  print('Capa: $capa');
}
Una List es como una columna de tu tabla de atributos — una colección ordenada de valores.
🗃️ Map — Atributos de un punto geográfico

Un Map es como una fila de la tabla de atributos de QGIS.

Dart
Map<String, dynamic> punto = {
  'nombre':   'Plaza Murillo',
  'latitud':  -16.4955,
  'longitud': -68.1336,
  'tipo':     'punto_interes',
  'activo':   true,
};

String nombre = punto['nombre'];  // 'Plaza Murillo'
punto['fecha'] = '2026-03-16';   // Agregar campo
Las claves son los nombres de columna y los valores son los datos de esa fila.
⚙️ Funciones — Operaciones reutilizables
Dart
// Sin retorno — solo ejecuta algo
void mostrarCoordenada(double lat, double lng) {
  print('Coordenada: $lat, $lng');
}

// Con retorno — devuelve un resultado
String formatear(double lat, double lng) {
  return '($lat, $lng)';
}

// Parámetro opcional con valor por defecto
void centrarMapa(double lat, double lng, {int zoom = 13}) {
  print('Centrando con zoom $zoom');
}

centrarMapa(-16.50, -68.15);            // zoom = 13
centrarMapa(-16.50, -68.15, zoom: 18);  // zoom = 18
Una función es como una herramienta de geoprocesamiento. Buffer recibe una geometría y distancia, y devuelve el resultado.
🔀 Condicionales — Tomar decisiones
Dart
int zoom = 15;

if (zoom > 14) {
  print('Vista detallada — nombres de calles');
} else if (zoom > 10) {
  print('Vista intermedia — carreteras principales');
} else {
  print('Vista general — límites departamentales');
}
En QGIS ya haces esto cuando defines reglas de simbología por escala. Misma lógica, diferente sintaxis.
🔗 Interpolación de Strings — Texto dinámico
Dart
String servidor = 'https://miservidor.com/geoserver';
String capa = 'municipios';

String url = '$servidor/wms?service=WMS&layers=$capa';
// https://miservidor.com/geoserver/wms?service=WMS&layers=municipios

🎯 Objetivo

Entender cómo Flutter construye interfaces, dominar los widgets esenciales y terminar con un mapa interactivo de OpenStreetMap en tu teléfono.

🧩 Todo en Flutter es un Widget

En QGIS construyes con capas apiladas. En Flutter, con widgets apilados y anidados.

QGIS vs Flutter — Comparación directa
Comparación
QGIS                          Flutter
─────────────────             ─────────────────
Proyecto (.qgz)         →     MaterialApp
Vista de mapa           →     Scaffold
Panel de capas          →     Column de widgets
Capa OpenStreetMap      →     TileLayer
Capa de puntos          →     MarkerLayer
Tabla de atributos      →     ListView
Barra de herramientas   →     AppBar + FloatingActionButton
Ya sabes construir interfaces — lo haces en QGIS con capas. Ahora hazlo en Flutter con la misma lógica.
📌 StatelessWidget vs StatefulWidget
StatelessWidget
No cambia después de crearse.
Ej: título de la app, ícono fijo.
StatefulWidget
Cambia mientras la app está abierta.
Ej: mapa (zoom), posición GPS.
🔄 setState — "Algo cambió, redibuja"
Dart
int _contadorPuntos = 0;

void _agregarPunto() {
  setState(() {
    _contadorPuntos = _contadorPuntos + 1;
  });
  // Le dice a Flutter: "actualiza la pantalla"
}

🏗️ Widgets Esenciales

Scaffold, Column, Row, Expanded
Dart
Scaffold(
  appBar: AppBar(title: Text('GeoCollect')),
  body: Column(
    children: [
      Expanded(          // Ocupa todo el espacio
        child: Text('Aquí va el mapa'),
      ),
      Container(          // Tamaño fijo
        height: 60,
        child: Row(       // Horizontal
          children: [
            Icon(Icons.location_on),
            Text('Lat: -16.50'),
          ],
        ),
      ),
    ],
  ),
)
💡
Expanded es clave: le dice al mapa "ocupa todo el espacio que queda". Sin él, Flutter no sabe cuánto espacio darle.

🗺️ Tu Primer Mapa Real — flutter_map

Paso 1 — Agregar dependencias

En pubspec.yaml, sección dependencies:

yaml
dependencies:
  flutter:
    sdk: flutter
  flutter_map: ^7.0.2
  latlong2: ^0.9.1
Terminal
flutter pub get
Instalar un paquete en Flutter es como instalar un plugin en QGIS.
Paso 2 — Permiso de Internet

En android/app/src/main/AndroidManifest.xml, antes de <application>:

XML
<uses-permission android:name="android.permission.INTERNET"/>
⚠️
Sin este permiso la app se abre pero el mapa aparece en blanco. Error #1 de esta clase.
Paso 3 — Código completo del mapa

Reemplaza todo el contenido de lib/main.dart:

Dart — lib/main.dart
import 'package:flutter/material.dart';
import 'package:flutter_map/flutter_map.dart';
import 'package:latlong2/latlong.dart';

void main() {
  runApp(const GeoCollectApp());
}

class GeoCollectApp extends StatelessWidget {
  const GeoCollectApp({super.key});

  @override
  Widget build(BuildContext context) {
    return MaterialApp(
      title: 'GeoCollect',
      debugShowCheckedModeBanner: false,
      theme: ThemeData(
        colorSchemeSeed: Colors.teal,
        useMaterial3: true,
      ),
      home: const MapaScreen(),
    );
  }
}

class MapaScreen extends StatefulWidget {
  const MapaScreen({super.key});
  @override
  State<MapaScreen> createState() => _MapaScreenState();
}

class _MapaScreenState extends State<MapaScreen> {
  // Cambia estas coordenadas por las de tu ciudad
  final LatLng _centro = const LatLng(-16.5, -68.15);

  @override
  Widget build(BuildContext context) {
    return Scaffold(
      appBar: AppBar(title: const Text('GeoCollect')),
      body: Column(children: [
        Expanded(
          child: FlutterMap(
            options: MapOptions(
              initialCenter: _centro,
              initialZoom: 13,
            ),
            children: [
              TileLayer(
                urlTemplate: 'https://tile.openstreetmap.org/{z}/{x}/{y}.png',
                userAgentPackageName: 'com.example.geo_collect',
              ),
              MarkerLayer(markers: [
                Marker(
                  point: _centro,
                  width: 40, height: 40,
                  child: const Icon(Icons.location_on,
                    color: Colors.red, size: 40),
                ),
              ]),
            ],
          ),
        ),
        Container(
          padding: const EdgeInsets.symmetric(
            horizontal: 16, vertical: 12),
          color: Colors.teal.shade700,
          child: Row(
            mainAxisAlignment: MainAxisAlignment.spaceBetween,
            children: [
              Text('📍 La Paz, Bolivia',
                style: const TextStyle(color: Colors.white)),
              Text('Zoom: 13.0',
                style: const TextStyle(color: Colors.white)),
            ],
          ),
        ),
      ]),
    );
  }
}

📖 Glosario — Clase 02

WidgetCualquier elemento visual en Flutter: texto, botón, mapa, contenedor.
StatelessWidget que no cambia una vez creado (título, ícono).
StatefulWidget que puede cambiar (mapa, datos GPS, contador).
setState()Notifica a Flutter que algo cambió y debe redibujar.
ScaffoldEstructura base de pantalla: AppBar + body + botones.
Column/RowOrganizan hijos vertical (Column) u horizontalmente (Row).
Expanded"Ocupa todo el espacio disponible". Esencial para el mapa.
FlutterMapWidget de mapa interactivo que renderiza tiles.
TileLayerCapa que carga las imágenes del mapa base (tiles).
MarkerLayerCapa de marcadores/puntos sobre el mapa.
TileImagen cuadrada que forma parte del mapa. Se descargan según zoom y posición.
pubspec.yamlArchivo donde declaras las dependencias del proyecto.
