🎯 Objetivo

Entender qué es un servidor GIS, cómo funciona el estándar WMS, y conectar tu app con capas de un GeoServer real.

🖥️ ¿Qué es GeoServer?

Un servidor que sirve mapas por internet

GeoServer es un programa que se instala en un servidor (computadora conectada a internet). Su trabajo es tomar datos geográficos (shapefiles, bases de datos PostGIS, imágenes GeoTIFF) y servirlos a cualquier aplicación que los pida — tu app Flutter, un sitio web, QGIS, o cualquier otro cliente.

Piensa en un restaurante:

🗄️
Datos
Shapefiles, PostGIS, GeoTIFF = ingredientes en la cocina
🖥️
GeoServer
La cocina que prepara los platos (imágenes de mapa)
📋
WMS
El menú: pides "dame esta capa en esta zona"
📱
Tu App
El cliente que pide y recibe el plato servido
💡
Tu app NO descarga los datos originales. Solo pide imágenes renderizadas del mapa. Es como pedir un plato sin entrar a la cocina. Los datos pesados se quedan en el servidor.
Ya hiciste esto antes: en QGIS cuando agregas una capa WMS (Capa → Agregar capa WMS/WMTS), QGIS se conecta al servidor y muestra las imágenes. Tu app Flutter hará exactamente lo mismo.

📋 ¿Qué es WMS?

Web Map Service — El estándar para pedir mapas

WMS es un estándar internacional (de la OGC — Open Geospatial Consortium) que define cómo pedirle imágenes de mapa a un servidor. Es como un idioma universal: cualquier servidor que hable WMS puede ser consumido por cualquier cliente que entienda WMS.

Cuando tu app pide una capa WMS, le dice al servidor:

"Dame la capa 'departamentos',
en la zona de latitud -17 a -16, longitud -69 a -67,
en formato PNG con fondo transparente,
usando el sistema de coordenadas EPSG:4326."

Y el servidor responde con una imagen PNG de esa zona con los datos dibujados.

Anatomía de una URL WMS

Una petición WMS es simplemente una URL con parámetros. Cada parámetro le dice algo al servidor:

URL WMS
https://miservidor.com/geoserver/wms?
  service=WMS            ← Tipo de servicio
  &request=GetMap         ← "Dame una imagen"
  &layers=geo:rios        ← Qué capa quiero
  &bbox=-69,-17,-67,-16   ← Zona geográfica
  &width=800&height=600   ← Tamaño en píxeles
  &srs=EPSG:4326          ← Sistema de coordenadas
  &format=image/png       ← Formato de imagen
  &transparent=true       ← Fondo transparente
💡
No necesitas construir esta URL a mano. flutter_map la construye automáticamente. Solo le das la URL base y los parámetros principales.

🔌 Conectar con GeoServer desde Flutter

WMSTileLayerOptions — El puente entre tu app y GeoServer

flutter_map tiene soporte nativo para WMS. Solo necesitas decirle la URL del servidor y el nombre de la capa:

Dart
// Agregar a los children de FlutterMap
TileLayer(
  wmsOptions: WMSTileLayerOptions(
    baseUrl: 'https://miservidor.com/geoserver/wms?',
    layers: ['workspace:departamentos'],
    transparent: true,
    format: 'image/png',
    version: '1.1.1',
    otherParameters: {
      'SRS': 'EPSG:4326',
    },
  ),
),
⚠️
El ? al final de baseUrl es OBLIGATORIO. flutter_map agrega los parámetros después de ese ?. Sin él, la URL queda mal formada y no carga nada.
¿Qué es "workspace:nombre_capa"?

En GeoServer, los datos se organizan en workspaces (espacios de trabajo). El nombre de una capa siempre tiene el formato workspace:nombre. Es como una ruta de carpeta: datos_bolivia:departamentos significa "la capa departamentos dentro del workspace datos_bolivia".

🗂️ Múltiples Capas WMS + Panel de Control

Toggle de capas con Drawer

En QGIS tienes un panel de capas con checkboxes. En Flutter, usamos un Drawer (panel lateral) con switches:

Dart
// Estado: qué capas están activas
Map<String, bool> _capasWMS = {
  'departamentos': true,
  'rios': false,
  'carreteras': false,
};

// En FlutterMap children — genera capas dinámicamente
..._capasWMS.entries.where((e) => e.value).map((e) {
  return TileLayer(
    wmsOptions: WMSTileLayerOptions(
      baseUrl: 'https://miservidor.com/geoserver/wms?',
      layers: ['workspace:${e.key}'],
      transparent: true,
      format: 'image/png',
    ),
  );
}).toList(),
¿Qué es un Drawer?

Un Drawer es un panel lateral que se desliza desde el borde izquierdo de la pantalla. En Android lo ves en muchas apps (Gmail, Google Maps). Es ideal para poner controles secundarios como la lista de capas, sin ocupar espacio permanente en pantalla.

Dart
// Drawer con switches para cada capa
Drawer(
  child: ListView(
    padding: const EdgeInsets.all(16),
    children: [
      const Text('Capas WMS',
        style: TextStyle(fontSize: 18, fontWeight: FontWeight.bold)),
      ..._capasWMS.entries.map((e) => SwitchListTile(
        title: Text(e.key),
        value: e.value,
        onChanged: (v) => setState(() => _capasWMS[e.key] = v),
      )).toList(),
    ],
  ),
)
Es exactamente el panel de capas de QGIS — checkboxes para activar/desactivar capas. Misma funcionalidad, diferente interfaz.

🔍 Filtros CQL — Consultar datos en el servidor

¿Qué es CQL?

CQL = Common Query Language. Es un lenguaje para filtrar datos en el servidor antes de que los envíe. En vez de descargar todos los municipios de Bolivia y filtrar en tu app, le dices al servidor: "solo mándame los de La Paz".

Si conoces SQL, CQL es muy similar a la cláusula WHERE:

SQL (QGIS/PostgreSQL)
WHERE depto = 'La Paz'
CQL (GeoServer)
CQL_FILTER=depto='La Paz'
💡
¿Por qué filtrar en el servidor? Porque es más eficiente. En vez de enviar 1000 municipios y que tu app descarte 900, el servidor solo envía los 100 que necesitas. Menos datos = más rápido.
Aplicar CQL en flutter_map
Dart
TileLayer(
  wmsOptions: WMSTileLayerOptions(
    baseUrl: 'https://miservidor.com/geoserver/wms?',
    layers: ['workspace:municipios'],
    transparent: true,
    format: 'image/png',
    otherParameters: {
      'CQL_FILTER': "departamento='La Paz'",
    },
  ),
),
Ejemplos de filtros CQL
CQL
// Por valor exacto de texto
departamento='La Paz'

// Por rango numérico
poblacion>100000

// Combinados con AND
departamento='La Paz' AND poblacion>50000

// Por proximidad geográfica (10 km de un punto)
DWITHIN(geometria, POINT(-68.15 -16.50), 10000, meters)
⚠️
Los nombres de campos deben coincidir exactamente con los del servidor. Si en GeoServer el campo se llama depto, no puedes poner departamento. Verifica los nombres en la previsualización de GeoServer.

👆 GetFeatureInfo — Consultar atributos al tocar

¿Qué es GetFeatureInfo?

Es otro tipo de petición WMS. En vez de pedir una imagen (GetMap), pides información del feature que está en un punto específico. Es como la herramienta "Identificar" de QGIS: haces clic en un polígono y ves sus atributos.

El flujo es:

1. El usuario toca el mapa → obtienes coordenadas (píxel)
2. Tu app construye una URL GetFeatureInfo con esas coordenadas
3. El servidor responde con los atributos en formato JSON
4. Tu app muestra los atributos en un diálogo

yaml — pubspec.yaml
dependencies:
  http: ^1.2.0   # ← NUEVO: para hacer peticiones HTTP
¿Qué es el paquete http?

http es un paquete que permite a tu app hacer peticiones a servidores web — como abrir una URL y leer la respuesta. Lo necesitamos porque GetFeatureInfo devuelve datos JSON (no una imagen como GetMap).

🎨 Leyenda WMS

El servidor genera la leyenda automáticamente

GeoServer puede generar una imagen de la leyenda de cualquier capa. Solo necesitas pedirla con el tipo de petición GetLegendGraphic:

Dart
String urlLeyenda(String capa) {
  return 'https://miservidor.com/geoserver/wms?'
    'service=WMS&version=1.1.1'
    '&request=GetLegendGraphic'
    '&layer=workspace:$capa'
    '&format=image/png';
}

// Mostrar como imagen en tu app
Image.network(urlLeyenda('departamentos'))
Es la leyenda del mapa — exactamente como la que ves en QGIS abajo de cada capa. Pero generada por el servidor.
