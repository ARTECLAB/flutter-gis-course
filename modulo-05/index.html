🎯 Objetivo

Crear un formulario profesional para capturar puntos de campo con atributos, validar los datos, y guardarlos de forma persistente en el teléfono.

📋 ¿Por qué necesitamos formularios?

Una coordenada sola no sirve de mucho

Capturar un punto GPS sin atributos es como tomar una foto sin descripción. En trabajo de campo necesitas registrar: ¿Qué tipo de punto es? ¿En qué estado está? ¿Alguna observación? Esos datos son los atributos.

En QGIS, cuando digitalizas un punto, se abre una ventana donde llenas los campos de la tabla de atributos. En tu app, se abre un formulario.

🧩 Widgets de Formulario

Form y GlobalKey — El contenedor y su control remoto

Form es un widget que agrupa todos los campos del formulario. GlobalKey<FormState> es como un control remoto: te permite validar todos los campos a la vez con un solo comando.

Dart
final _formKey = GlobalKey<FormState>();

Form(
  key: _formKey,
  child: Column(children: [
    // Aquí van los campos...
  ]),
)

// Validar TODO el formulario con una sola línea:
if (_formKey.currentState!.validate()) {
  // Todos los campos son válidos — guardar
}
TextFormField — Campo de texto con validación

Es un campo donde el usuario escribe texto. El validator verifica si el dato es válido y muestra un mensaje de error rojo si no lo es.

Dart
final _nombreCtrl = TextEditingController();

TextFormField(
  controller: _nombreCtrl,
  decoration: const InputDecoration(
    labelText: 'Nombre del punto',
    hintText: 'Ej: Poste eléctrico #42',
    prefixIcon: Icon(Icons.label),
    border: OutlineInputBorder(),
  ),
  validator: (valor) {
    if (valor == null || valor.isEmpty) {
      return 'El nombre es obligatorio';
    }
    return null; // null = válido
  },
)
💡
¿Qué es un TextEditingController? Es un objeto que "controla" un campo de texto. Te permite leer lo que escribió el usuario (_nombreCtrl.text) y también escribir valores por código. Cada campo de texto necesita su propio controller.
DropdownButtonFormField — Selector de opciones

Un menú desplegable con opciones predefinidas. Ideal para campos como "tipo" o "categoría" donde las opciones son fijas.

Dart
String? _tipo; // null = nada seleccionado

DropdownButtonFormField<String>(
  value: _tipo,
  decoration: const InputDecoration(
    labelText: 'Tipo de punto',
    border: OutlineInputBorder(),
  ),
  items: ['Poste', 'Árbol', 'Edificio', 'Muestra']
    .map((t) => DropdownMenuItem(value: t, child: Text(t)))
    .toList(),
  onChanged: (v) => setState(() => _tipo = v),
  validator: (v) => v == null ? 'Selecciona un tipo' : null,
)
RadioListTile — Opciones excluyentes

Cuando solo se puede elegir UNA opción de un grupo (bueno/regular/malo).

Dart
String _estado = 'Bueno';

...['Bueno', 'Regular', 'Malo'].map((e) =>
  RadioListTile(
    title: Text(e),
    value: e,
    groupValue: _estado,
    onChanged: (v) => setState(() => _estado = v!),
  ),
)

💾 Almacenamiento Local — SharedPreferences

¿Por qué guardar datos localmente?

En campo muchas veces no hay internet. Los datos deben guardarse en el teléfono y persistir aunque cierres la app. SharedPreferences guarda datos como texto (JSON) en el almacenamiento interno del teléfono.

💡
¿Qué es JSON? Es un formato de texto para representar datos estructurados. Tu lista de puntos se convierte en texto JSON para guardarse, y se reconvierte a lista de Dart al cargarse. Es como guardar una tabla de Excel como archivo CSV — los datos se convierten a texto.
yaml — pubspec.yaml
dependencies:
  shared_preferences: ^2.2.0
Dart
import 'package:shared_preferences/shared_preferences.dart';
import 'dart:convert';

// GUARDAR — convierte lista a JSON y guarda
Future<void> _guardarPuntos() async {
  final prefs = await SharedPreferences.getInstance();
  final texto = json.encode(_puntos);
  await prefs.setString('puntos_campo', texto);
}

// CARGAR — lee JSON y convierte a lista
Future<void> _cargarPuntos() async {
  final prefs = await SharedPreferences.getInstance();
  final texto = prefs.getString('puntos_campo');
  if (texto != null) {
    setState(() {
      _puntos = (json.decode(texto) as List)
        .map((e) => Map<String, dynamic>.from(e))
        .toList();
    });
  }
}
⚠️
Llama _guardarPuntos() después de cada cambio (agregar, editar, eliminar). Si solo modificas la lista en memoria pero no guardas, los cambios se pierden al cerrar la app.

📋 Lista de Registros con ListView.builder

¿Qué es ListView.builder?

Es un widget que crea una lista scrolleable. .builder significa que construye cada elemento bajo demanda — solo los que están visibles en pantalla. Es eficiente aunque tengas 1000 registros.

Dart
ListView.builder(
  itemCount: _puntos.length,
  itemBuilder: (ctx, i) {
    final p = _puntos[i];
    return ListTile(
      leading: Icon(Icons.location_on),
      title: Text(p['nombre']),
      subtitle: Text('${p['tipo']} · ${p['estado']}'),
    );
  },
)
Dismissible — Eliminar deslizando

Envuelve cada elemento en Dismissible para que el usuario pueda eliminarlo deslizando hacia un lado. Es el patrón estándar de Android para borrar.

Dart
Dismissible(
  key: Key(p['id'].toString()),
  direction: DismissDirection.endToStart,
  background: Container(
    color: Colors.red,
    alignment: Alignment.centerRight,
    padding: const EdgeInsets.only(right: 20),
    child: const Icon(Icons.delete, color: Colors.white),
  ),
  onDismissed: (_) {
    setState(() => _puntos.removeAt(i));
    _guardarPuntos();
  },
  child: ListTile(/* ... */),
)

🧭 Navegación entre Pantallas

Navigator — El sistema de pantallas de Flutter

Tu app tiene varias pantallas: Mapa, Formulario, Lista. Navigator es el sistema que controla la navegación entre ellas. Funciona como un stack (pila) de cartas:

push
Pone una pantalla nueva encima.
"Ir al formulario"
pop
Quita la pantalla de encima.
"Volver atrás"
Dart
// Ir al formulario
Navigator.push(context, MaterialPageRoute(
  builder: (_) => FormularioPunto(posicion: _gps!),
));

// Volver atrás (dentro del formulario)
Navigator.pop(context);

📱 BottomNavigationBar — Mapa ↔ Lista

Barra inferior con pestañas

Como Google Maps tiene "Explorar" y "Guardados" abajo, tu app tendrá "Mapa" y "Datos". El usuario toca la pestaña y cambia de vista.

Dart
int _pantalla = 0;

Scaffold(
  body: IndexedStack(
    index: _pantalla,
    children: [
      _buildMapa(),   // índice 0
      _buildLista(),  // índice 1
    ],
  ),
  bottomNavigationBar: BottomNavigationBar(
    currentIndex: _pantalla,
    onTap: (i) => setState(() => _pantalla = i),
    items: const [
      BottomNavigationBarItem(
        icon: Icon(Icons.map), label: 'Mapa'),
      BottomNavigationBarItem(
        icon: Icon(Icons.list), label: 'Datos'),
    ],
  ),
)
💡
¿Qué es IndexedStack? Es un widget que tiene varios hijos pero solo muestra uno a la vez (el del índice indicado). Los demás están ocultos pero mantienen su estado — el mapa no se reinicia al cambiar a la lista.
