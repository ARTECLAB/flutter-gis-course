📡 ¿Cómo funciona el GPS en una app?

El GPS es un sensor de hardware

Tu teléfono tiene un chip GPS adentro. Este chip recibe señales de satélites en el espacio y calcula tu posición exacta (latitud y longitud) mediante trilateración — necesita al menos 4 satélites para obtener una posición 3D. Es el mismo principio que un GPS Garmin de campo, pero integrado en el teléfono.

Para que tu app acceda al GPS necesita 3 cosas obligatorias:

1️⃣
Permiso en el código
Declarar en AndroidManifest.xml que la app necesita ubicación.
2️⃣
Permiso del usuario
El usuario debe aceptar "¿Permitir acceso a ubicación?"
3️⃣
GPS encendido
El sensor GPS del teléfono debe estar activado.
⚠️
Si falta cualquiera de estas 3 cosas, la app no puede obtener coordenadas. Tu código debe verificar las 3 condiciones y mostrar un mensaje claro al usuario si algo falta.
Es como preparar equipo para salir a campo: primero verificas que el GPS tenga baterías (sensor encendido), que tenga señal de satélites (servicio habilitado), y que tengas la llave del equipo (permiso). Sin cualquiera de los tres, no trabajas.

📦 Instalar y configurar geolocator

¿Qué es geolocator?

Es el paquete estándar de Flutter para geolocalización. Funciona en Android, iOS, Web, Windows, macOS y Linux. Te da acceso al GPS sin escribir código nativo — tú le pides la posición y él maneja toda la complejidad del hardware.

Usamos la versión 14.0.2 (la más reciente). Nos permite:

  • getCurrentPosition() — Lectura única (como tomar un waypoint)
  • getPositionStream() — Flujo continuo (como un tracklog)
  • getLastKnownPosition() — Última posición guardada (instantánea)
  • distanceBetween() — Distancia entre 2 puntos (Haversine)
  • bearingBetween() — Rumbo/azimut entre 2 puntos
  • Verificación y solicitud de permisos
  • openAppSettings() / openLocationSettings() — Redirigir a configuración

🔐 Configurar Permisos en Android

¿Qué son los permisos de Android?

Android protege la privacidad del usuario. Si una app quiere acceder a sensores sensibles (GPS, cámara, micrófono), primero debe declararlo en su código y luego pedirle permiso al usuario en tiempo de ejecución.

Existen dos niveles de precisión de ubicación:

ACCESS_FINE_LOCATION
GPS satelital. Precisión ~3-10 metros.
En Android: PRIORITY_HIGH_ACCURACY
Es el que necesitamos para GIS.
ACCESS_COARSE_LOCATION
WiFi y antenas celulares. ~100 metros.
En Android: PRIORITY_LOW_POWER
No suficiente para trabajo GIS.
💡
La documentación oficial dice: "cuando configuras ambos permisos, el plugin usa ACCESS_FINE_LOCATION". En Android 12+, el usuario puede elegir "ubicación aproximada" — en ese caso recibirás COARSE aunque pidas FINE. Es decisión del usuario.
Paso 4 — Declarar los permisos en AndroidManifest.xml

Abre android/app/src/main/AndroidManifest.xml y agrega antes de <application>:

XML — AndroidManifest.xml
<!-- Permisos de red (tiles del mapa — ya debería estar) -->
<uses-permission android:name="android.permission.INTERNET" />

<!-- Permisos de ubicación GPS -->
<uses-permission android:name="android.permission.ACCESS_FINE_LOCATION"/>
<uses-permission android:name="android.permission.ACCESS_COARSE_LOCATION"/>
⚠️
Después de cambiar AndroidManifest.xml necesitas hacer un restart completo (flutter run de nuevo). Hot Reload NO aplica cambios de archivos nativos — solo actualiza código Dart.
📋
Nota avanzada: Para seguimiento en segundo plano (app minimizada) se necesitan permisos adicionales: ACCESS_BACKGROUND_LOCATION (Android 10+) y FOREGROUND_SERVICE_LOCATION (Android 14+). No los necesitamos en este módulo, pero existen para apps profesionales de campo.
Paso 5 — Verificar y solicitar permisos en Dart

Esta función verifica las 3 condiciones necesarias. Es exactamente el patrón que recomienda la documentación oficial de geolocator:

Dart — gps_service.dart
import 'package:geolocator/geolocator.dart';

/// Verifica las 3 condiciones para usar GPS.
/// Retorna un String con el error, o null si todo está listo.
Future<String?> verificarPermisos() async {
  bool servicioActivo;
  LocationPermission permiso;

  // Condición 1: ¿El GPS del teléfono está encendido?
  servicioActivo = await Geolocator.isLocationServiceEnabled();
  if (!servicioActivo) {
    return '⚠️ El GPS está apagado. Actívalo en configuración.';
  }

  // Condición 2: ¿La app tiene permiso del usuario?
  permiso = await Geolocator.checkPermission();
  if (permiso == LocationPermission.denied) {
    // Solicitar permiso al usuario
    permiso = await Geolocator.requestPermission();
    if (permiso == LocationPermission.denied) {
      return '⚠️ Permiso de ubicación denegado.';
    }
  }

  // Condición 3: ¿Denegado permanentemente?
  if (permiso == LocationPermission.deniedForever) {
    return '⚠️ Permiso denegado permanentemente. '
           'Ve a Configuración → Apps → Permisos.';
  }

  return null; // ¡Todo listo! ✅
}
💡
¿Y si el usuario denegó permanentemente? Android NO vuelve a mostrar el diálogo de permiso. Puedes redirigir al usuario a la pantalla de configuración con Geolocator.openAppSettings() o a la configuración de ubicación con Geolocator.openLocationSettings().
Es como verificar tu equipo antes de salir a campo: ¿GPS cargado? ¿Satélites conectados? ¿Precisión aceptable? Si algo falla, no empiezas a levantar datos.

📍 Obtener Tu Posición Actual

getCurrentPosition() — Tomar un waypoint

Es una función que le pregunta al GPS del teléfono: "¿dónde estoy ahora?". El GPS tarda unos segundos en responder (la primera vez puede tardar 5-15 segundos mientras conecta con satélites). Por eso usamos async/await — para esperar la respuesta sin bloquear la app.

💡
¿Qué es async/await? Imagina que pides pizza por teléfono. async marca una función que toma tiempo. await espera a que llegue el resultado antes de continuar. Mientras esperas, la app sigue funcionando (puedes mover el mapa, tocar botones). Sin await, tu código continuaría ejecutándose sin tener la posición todavía.
⚠️
API actualizada en geolocator 14: El parámetro desiredAccuracy fue reemplazado por locationSettings. El código correcto usa LocationSettings(accuracy: LocationAccuracy.high). Si ves tutoriales con desiredAccuracy: directo, es la API vieja.
Dart — API oficial geolocator 14
import 'package:geolocator/geolocator.dart';
import 'package:latlong2/latlong.dart';

LatLng? _posicionGPS;     // Posición actual (null si no hay)
bool _cargandoGPS = false; // ¿Estamos esperando al GPS?

Future<void> _obtenerUbicacion() async {
  // Verificar permisos primero
  final error = await verificarPermisos();
  if (error != null) return; // Mostrar error al usuario

  setState(() => _cargandoGPS = true);

  // Configuración de la lectura GPS
  const LocationSettings locationSettings = LocationSettings(
    accuracy: LocationAccuracy.high,  // GPS satelital
  );

  // Obtener posición actual
  final Position pos = await Geolocator.getCurrentPosition(
    locationSettings: locationSettings,
  );

  setState(() {
    _posicionGPS = LatLng(pos.latitude, pos.longitude);
    _cargandoGPS = false;
  });

  // Centrar el mapa en mi posición
  _mapController.move(_posicionGPS!, 16.0);
}
LocationAccuracy — Niveles de precisión

El enum LocationAccuracy controla qué tan precisa será la lectura GPS y cuánta batería consume:

Nivel Android (Prioridad) Precisión aprox. Uso
lowest PRIORITY_PASSIVE ~3 km Solo datos pasivos, sin consumo extra
low PRIORITY_LOW_POWER ~1 km Solo redes, ahorra batería
medium PRIORITY_BALANCED ~100 m Balance precisión/batería
high PRIORITY_HIGH_ACCURACY ~3-10 m ✅ El que usamos para GIS
best Máxima disponible ~3 m Máxima precisión, más batería
bestForNavigation + sensores extra ~1-3 m Usa giroscopio, acelerómetro
¿Qué es el signo ? y ! en Dart?

LatLng? (con signo de pregunta) significa que esta variable puede ser null — puede no tener valor todavía. Al iniciar la app, no tenemos posición GPS, así que empieza como null. Después de obtener el GPS, le asignamos un valor real.

_posicionGPS! (con signo de exclamación) le dice a Dart: "estoy seguro de que NO es null, úsalo". Solo usamos ! después de verificar con if (_posicionGPS != null).

getLastKnownPosition() — Dato rápido

Mientras el GPS busca satélites (puede tardar segundos), puedes obtener la última posición guardada en el dispositivo de forma instantánea:

Dart
// Posición instantánea (puede ser de hace minutos u horas)
Position? ultimaPos = await Geolocator.getLastKnownPosition();
if (ultimaPos != null) {
  // Mostrar mientras esperamos el GPS real
  _posicionGPS = LatLng(ultimaPos.latitude, ultimaPos.longitude);
}
💡
Patrón profesional: Primero muestra getLastKnownPosition() para dar feedback inmediato, y luego actualiza con getCurrentPosition() cuando llegue la lectura real del GPS.

🔄 Seguimiento GPS en Tiempo Real

¿Qué es un Stream?

En la clase anterior, pedimos la posición una vez con getCurrentPosition(). Pero en campo necesitas que la posición se actualice automáticamente mientras caminas.

Un Stream es un flujo continuo de datos. Piensa en la diferencia:

getCurrentPosition()
Sacas UNA foto → recibes UNA imagen.
Útil para: "¿dónde estoy ahora?"
getPositionStream()
Enciendes una cámara de video → flujo continuo.
Útil para: "sígueme mientras camino"
Dart — API oficial geolocator 14
import 'dart:async';
import 'package:geolocator/geolocator.dart';

StreamSubscription<Position>? _stream;
bool _siguiendo = false;

void _iniciarSeguimiento() async {
  final error = await verificarPermisos();
  if (error != null) return;

  // Configuración: precisión alta, actualizar cada 5 metros
  const LocationSettings locationSettings = LocationSettings(
    accuracy: LocationAccuracy.high,
    distanceFilter: 5,  // metros mínimos de movimiento
  );

  _stream = Geolocator.getPositionStream(
    locationSettings: locationSettings,
  ).listen((Position pos) {
    setState(() {
      _posicionGPS = LatLng(pos.latitude, pos.longitude);
    });
  });

  setState(() => _siguiendo = true);
}

void _detenerSeguimiento() {
  _stream?.cancel();    // Cerrar el flujo
  _stream = null;
  setState(() => _siguiendo = false);
}
💡
distanceFilter: 5 significa "solo notifícame cuando me haya movido al menos 5 metros". Sin esto, el GPS enviaría posiciones cada segundo aunque estés quieto — gastando batería sin razón. Para logística usa 100, para campo 5-10.
⚠️
Siempre cancela el Stream cuando ya no lo necesitas. Si no lo cancelas, sigue consumiendo GPS y batería en segundo plano. Agrega _stream?.cancel() en el método dispose() de tu widget.
¿Qué es dispose()?

dispose() es un método que Flutter llama automáticamente cuando un widget se destruye (al cerrar la pantalla, al navegar a otra página). Es el lugar correcto para limpiar recursos: cancelar streams, detener animaciones, liberar controladores.

Dart
@override
void dispose() {
  _stream?.cancel();        // Detener GPS
  _gpsService.dispose();   // Si usas un servicio
  super.dispose();
}
Es como apagar el GPS cuando terminas el trabajo de campo. Si lo dejas encendido, se le acaba la batería sin motivo.

📌 Capturar Puntos GPS

Registrar waypoints con timestamp

Capturar un punto significa tomar la posición GPS del momento y guardarla en una lista con fecha y hora. Es exactamente como marcar un waypoint en un GPS Garmin.

Dart — Modelo de punto capturado
class PuntoCapturado {
  final int id;
  final LatLng posicion;
  final DateTime fecha;
  String nota;

  PuntoCapturado({
    required this.id,
    required this.posicion,
    required this.fecha,
    this.nota = '',
  });
}
Dart — Capturar punto
List<PuntoCapturado> _puntosCapturados = [];

void _capturarPuntoGPS() {
  if (_posicionGPS == null) return;

  final punto = PuntoCapturado(
    id: DateTime.now().millisecondsSinceEpoch,
    posicion: _posicionGPS!,
    fecha: DateTime.now(),
  );

  setState(() {
    _puntosCapturados.add(punto);
  });
}
En QGIS, cuando digitalizas un punto haces clic en el mapa y se agrega a la capa con sus atributos. Aquí el "clic" es presionar un botón, y los atributos son las coordenadas GPS + la fecha automática.

📐 Cálculo de Distancias y Rumbos

distanceBetween() — Distancia entre dos puntos

La Tierra es una esfera, así que la distancia entre dos coordenadas no es una línea recta euclidiana. Geolocator usa la fórmula de Haversine que calcula la distancia sobre la superficie esférica.

Dart — API oficial
// Distancia entre dos puntos (en metros)
double metros = Geolocator.distanceBetween(
  -16.4955, -68.1336,  // Plaza Murillo (lat, lon)
  -16.4978, -68.1369,  // Otro punto   (lat, lon)
);
print(metros); // ~420.3 metros

// Formato legible
String formatear(double m) {
  if (m < 1000) return '${m.toStringAsFixed(0)} m';
  return '${(m / 1000).toStringAsFixed(2)} km';
}
En QGIS usas la herramienta de medir distancia (clic, clic). Aquí la calculas por código con las mismas coordenadas. El resultado es idéntico porque ambos usan Haversine.
bearingBetween() — Rumbo / Azimut

Además de la distancia, geolocator puede calcular el bearing (rumbo o azimut) entre dos puntos — el ángulo respecto al norte geográfico:

Dart — API oficial
// Bearing entre dos puntos (en grados)
double bearing = Geolocator.bearingBetween(
  -16.4955, -68.1336,  // Punto A
  -16.4978, -68.1369,  // Punto B
);
// bearing ≈ 228° (suroeste)
// 0°=Norte, 90°=Este, 180°=Sur, 270°=Oeste
🧭
El bearing es como la lectura de una brújula: te dice en qué dirección está el punto B desde el punto A. Muy útil para navegación en campo y orientación.
Distancia total de un recorrido

Para calcular la distancia total de un track (varios puntos), sumamos la distancia entre cada par consecutivo:

Dart
/// Distancia total de una lista de puntos (track)
double distanciaTotal(List<LatLng> puntos) {
  double total = 0;
  for (int i = 1; i < puntos.length; i++) {
    total += Geolocator.distanceBetween(
      puntos[i - 1].latitude, puntos[i - 1].longitude,
      puntos[i].latitude, puntos[i].longitude,
    );
  }
  return total;
}
Es la distancia real del recorrido (como el cuentakilómetros de un auto), no la distancia en línea recta entre el primer y último punto.

StreamSubscriptionObjeto que representa tu "conexión" al Stream. Lo necesitas para cancelar (.cancel()) el flujo cuando ya no lo usas.

cancel()Cierra la suscripción al Stream. Deja de recibir datos y libera el recurso (GPS, batería).
dispose()Método del ciclo de vida de un StatefulWidget. Flutter lo llama cuando el widget se destruye. Lugar obligatorio para limpiar recursos.
distanceFilterDistancia mínima (en metros) que debes moverte para que el Stream emita una nueva posición. Ahorra batería y reduce ruido GPS.

BearingÁngulo en grados desde el norte geográfico hacia un punto destino. 0°=Norte, 90°=Este, 180°=Sur, 270°=Oeste. También llamado rumbo o azimut.
