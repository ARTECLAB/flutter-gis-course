📡 ¿Cómo funciona el GPS en una app?

El GPS es un sensor de hardware

Tu teléfono tiene un chip GPS adentro. Este chip recibe señales de satélites en el espacio y calcula tu posición exacta (latitud y longitud). Es el mismo principio que un GPS Garmin que se usa en campo, pero integrado en el teléfono.

Para que tu app acceda al GPS necesita 3 cosas obligatorias:

1️⃣
Permiso en el código
Declarar en AndroidManifest.xml que la app necesita ubicación.
2️⃣
Permiso del usuario
El usuario debe aceptar "¿Permitir acceso a ubicación?"
3️⃣
GPS encendido
El GPS del teléfono debe estar activado.
⚠️
Si falta cualquiera de estas 3 cosas, la app no puede obtener coordenadas. Tu código debe verificar las 3 y mostrar un mensaje claro al usuario si algo falta.
Es como conectar un GPS Garmin en campo: primero lo enciendes, luego esperas que conecte con satélites, y finalmente obtienes las coordenadas. En el teléfono es igual pero más rápido.

🔐 Configurar Permisos en Android

¿Qué son los permisos de Android?

Android protege la privacidad del usuario. Si una app quiere acceder a la cámara, GPS, contactos, etc., primero debe declararlo en su código y luego pedirle permiso al usuario. Es como pedir autorización antes de entrar a un área restringida.

Hay dos niveles de precisión de GPS:

ACCESS_FINE_LOCATION
GPS preciso por satélites. Precisión de ~3 metros. Es el que necesitamos para GIS.
ACCESS_COARSE_LOCATION
Ubicación aproximada por WiFi y antenas. ~100 metros. No suficiente para GIS.
Paso 1 — Declarar los permisos

Abre android/app/src/main/AndroidManifest.xml y agrega antes de <application>:

XML — AndroidManifest.xml
<uses-permission android:name="android.permission.ACCESS_FINE_LOCATION"/>
<uses-permission android:name="android.permission.ACCESS_COARSE_LOCATION"/>
⚠️
Después de cambiar AndroidManifest.xml necesitas hacer un restart completo de la app (flutter run de nuevo). Hot Reload NO funciona para cambios en AndroidManifest — solo actualiza archivos Dart.
Paso 2 — Verificar permisos en Dart

Esta función verifica las 3 condiciones necesarias y muestra un mensaje claro si algo falta:

Dart
import 'package:geolocator/geolocator.dart';

/// Verifica las 3 condiciones para usar GPS.
/// Retorna true si todo está listo.
Future<bool> _verificarPermisos() async {

  // Condición 1: ¿El GPS del teléfono está encendido?
  bool gpsEncendido = await Geolocator.isLocationServiceEnabled();
  if (!gpsEncendido) {
    // GPS apagado — no podemos continuar
    return false;
  }

  // Condición 2: ¿La app tiene permiso del usuario?
  LocationPermission permiso = await Geolocator.checkPermission();

  if (permiso == LocationPermission.denied) {
    // No tiene permiso — solicitarlo al usuario
    permiso = await Geolocator.requestPermission();
    if (permiso == LocationPermission.denied) {
      return false; // Usuario dijo que no
    }
  }

  // Condición 3: ¿Fue denegado permanentemente?
  if (permiso == LocationPermission.deniedForever) {
    // El usuario debe ir a Configuración manualmente
    return false;
  }

  return true; // ¡Todo listo!
}
Es como verificar tu equipo antes de salir a campo: ¿GPS cargado? ¿Satélites conectados? ¿Precisión aceptable? Si algo falla, no empiezas a levantar datos.

📍 Obtener Tu Posición Actual

¿Qué es getCurrentPosition?

Es una función que le pregunta al GPS del teléfono: "¿dónde estoy ahora?". El GPS tarda unos segundos en responder (la primera vez puede tardar 5-15 segundos mientras conecta con satélites). Por eso usamos async/await — para esperar la respuesta sin bloquear la app.

💡
¿Qué es async/await? Imagina que pides pizza por teléfono. async significa "esta acción toma tiempo". await significa "espera a que llegue la pizza antes de continuar". Mientras esperas, la app sigue funcionando (puedes mover el mapa, tocar botones).
Dart
LatLng? _posicionGPS;     // Posición actual (null si no hay)
bool _cargandoGPS = false; // ¿Estamos esperando al GPS?

Future<void> _obtenerUbicacion() async {
  // Primero verificar permisos
  bool listo = await _verificarPermisos();
  if (!listo) return;

  setState(() => _cargandoGPS = true);

  // Pedir posición al GPS
  Position pos = await Geolocator.getCurrentPosition(
    desiredAccuracy: LocationAccuracy.high,
  );

  setState(() {
    _posicionGPS = LatLng(pos.latitude, pos.longitude);
    _cargandoGPS = false;
  });

  // Centrar el mapa en mi posición
  _mapController.move(_posicionGPS!, 16.0);
}
¿Qué es el signo ? en LatLng?

LatLng? (con signo de pregunta) significa que esta variable puede ser null — es decir, puede no tener valor todavía. Al iniciar la app, no tenemos posición GPS, así que empieza como null. Después de obtener el GPS, le asignamos un valor real.

_posicionGPS! (con signo de exclamación) le dice a Dart: "sé que NO es null en este momento, úsalo sin miedo". Solo usamos ! cuando ya verificamos que tiene valor.

🔄 Seguimiento GPS en Tiempo Real

¿Qué es un Stream?

En la clase anterior, pedimos la posición una vez con getCurrentPosition(). Pero en campo necesitas que la posición se actualice automáticamente mientras caminas.

Un Stream es un flujo continuo de datos. Piensa en la diferencia:

getCurrentPosition()
Pides una foto → recibes UNA imagen.
Útil para: "¿dónde estoy ahora?"
getPositionStream()
Enciendes una cámara de video → flujo continuo.
Útil para: "sígueme mientras camino"
Dart
import 'dart:async';

StreamSubscription<Position>? _stream;
bool _siguiendo = false;

void _iniciarSeguimiento() async {
  bool ok = await _verificarPermisos();
  if (!ok) return;

  setState(() => _siguiendo = true);

  const config = LocationSettings(
    accuracy: LocationAccuracy.high,
    distanceFilter: 5,  // Actualiza cada 5 metros
  );

  _stream = Geolocator.getPositionStream(
    locationSettings: config,
  ).listen((Position pos) {
    setState(() {
      _posicionGPS = LatLng(pos.latitude, pos.longitude);
    });
  });
}

void _detenerSeguimiento() {
  _stream?.cancel();    // Cerrar el flujo
  setState(() => _siguiendo = false);
}
💡
distanceFilter: 5 significa "solo dame una nueva posición cuando me haya movido al menos 5 metros". Sin esto, el GPS enviaría posiciones cada segundo aunque estés quieto, gastando batería.
⚠️
Siempre cancela el Stream cuando ya no lo necesitas. Si no lo cancelas, sigue consumiendo GPS y batería en segundo plano. Agrega _stream?.cancel() en el método dispose() de tu widget.
¿Qué es dispose()?

dispose() es un método que Flutter llama automáticamente cuando un widget se destruye (por ejemplo, al cerrar la pantalla). Es el lugar correcto para limpiar recursos: cancelar streams, detener animaciones, liberar controladores.

Dart
@override
void dispose() {
  _stream?.cancel();  // Detener GPS
  super.dispose();
}

📌 Capturar Puntos GPS

Registrar la posición actual con timestamp

Capturar un punto significa tomar la posición GPS del momento y guardarla en una lista con fecha y hora. Es como marcar un waypoint en un GPS Garmin.

Dart
List<Map<String, dynamic>> _puntos = [];

void _capturarPunto() {
  if (_posicionGPS == null) return;

  final punto = {
    'lat':   _posicionGPS!.latitude,
    'lng':   _posicionGPS!.longitude,
    'fecha': DateTime.now().toIso8601String(),
    'pos':   LatLng(_posicionGPS!.latitude,
                   _posicionGPS!.longitude),
  };

  setState(() {
    _puntos.add(punto);
  });
}

📐 Cálculo de Distancias

¿Cómo se calcula la distancia entre dos coordenadas?

La Tierra es una esfera, así que la distancia entre dos puntos no es una línea recta simple. Se usa la fórmula de Haversine que tiene en cuenta la curvatura de la Tierra. La buena noticia: geolocator ya incluye esta fórmula.

Dart
// Distancia entre dos puntos (en metros)
double metros = Geolocator.distanceBetween(
  -16.4955, -68.1336,  // Punto A
  -16.4978, -68.1369,  // Punto B
);
print(metros); // ~420.3 metros

// Formato legible
String formatear(double m) {
  if (m < 1000) return '${m.toStringAsFixed(0)} m';
  return '${(m / 1000).toStringAsFixed(2)} km';
}
En QGIS usas la herramienta de medir distancia (clic, clic). Aquí la calculas por código con las mismas coordenadas. El resultado es idéntico.

cancel()Cierra el Stream. Deja de recibir datos y libera el recurso (GPS, batería).
dispose()Método de Flutter que se ejecuta cuando el widget se destruye. Lugar para limpiar recursos.
distanceFilterDistancia mínima (en metros) que debes moverte para recibir una nueva posición del Stream.
HaversineFórmula matemática para calcular distancias sobre una esfera. Tiene en cuenta la curvatura terrestre.
WaypointPunto marcado intencionalmente con coordenadas GPS. Lo que hacemos al "capturar" un punto.
TimestampMarca de fecha y hora exacta. Cada punto capturado incluye cuándo fue registrado.
